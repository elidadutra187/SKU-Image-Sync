Encontre ajuda para usar o SKU Image Sync.

Guia Rapido

1. Configuracao Inicial

Copie o arquivo .env.example para .env e preencha:

NUVEMSHOP_STORE_ID=seu_id_da_loja
NUVEMSHOP_ACCESS_TOKEN=seu_token
NUVEMSHOP_USER_AGENT=SKU Image Sync (seu@email.com)

2. Estrutura de Pastas

Organize suas imagens assim:

Fotos/
├── SKU001/
│   ├── 1.jpg
│   ├── 2.jpg
├── SKU002/
│   ├── 1.png
│   ├── 2.webp

3. Executando

Via interface web:

npm start
# Acesse http://localhost:3000

Via linha de comando:

# Simular (dry-run)
node sync-images.js --dry-run

# Adicionar novas
node sync-images.js --mode add

# Sincronizar
node sync-images.js --mode sync

# Substituir todas
node sync-images.js --mode replace

FAQ - Perguntas Frequentes

Como obter o token de acesso?

Acesse o painel de Apps da sua loja Nuvemshop ou crie um app em partners.nuvemshop.com.br. O token precisa ter permissao de leitura e escrita em produtos.

Quais formatos de imagem sao suportados?

JPG, JPEG, PNG, GIF e WebP. O tamanho maximo por imagem e 10MB.

O que faz cada modo?

  • Add: Apenas adiciona imagens novas
  • Sync: Adiciona novas e atualiza alteradas
  • Replace: Remove todas e reenvia da pasta

Como testar sem alterar a loja?

Use o modo Dry Run. Ele simula toda a sincronizacao e gera um relatorio, mas nao faz nenhuma alteracao real.

Onde fica o relatorio?

Um arquivo CSV e gerado na pasta do projeto com o nome relatorio-sync-[timestamp].csv.

A sincronizacao parou no meio. E agora?

Execute novamente com o modo sync. O arquivo de estado (.nuvemshop-sync-state.json) guarda o progresso e evita reenviar imagens ja sincronizadas.
